Mobile support for the replay stage (`components/match/ReplayViewer.vue` + `ReplayChrome.vue` + `Replay3DLite.vue`):

- **Landscape gate**: ReplayViewer keys everything off `(pointer: coarse)` = `mobileChrome` (true on phones AND tablets, false on a mouse/trackpad laptop) plus `(orientation: portrait)`. `needsRotate = coarse && portrait` shows a full-cover "rotate your device" overlay (z-[40]). No width threshold â€” tablets in landscape are >820px so a width gate would miss them.
- **Collapsible scoreboard**: ReplayChrome gained `mobile`, `showScoreboard`, `onToggleScoreboard` props. ReplayViewer owns `chromeScoreboardOpen` (starts collapsed on any touch device). The toggle is the first button in `.bp-sb-tools` (that strip stays mounted; `.sb-collapsed` slides it to the edge); only `.bp-score-panel` is `v-show`-hidden. The avatars/trace/deaths toggles are MAP controls so they stay visible. NOTE: the old `showScoreboard` ref + `scoreboardReserve` + the floating `.bp-score-panel` `v-if="false"` cluster in ReplayViewer are dead code â€” don't reuse them.
- **`.bp-chrome.is-mobile`** scoped CSS shrinks HUD / left column / scoreboard / transport for landscape phones.
- **three.js touch** (`Replay3DLite.vue`): on `(pointer: coarse)` it lowers pixelRatio cap to 1.5 + disables AA, lets OrbitControls own gestures (`enableZoom=true`, `touches: ONE=PAN, TWO=DOLLY_ROTATE` â€” one finger drags around, two fingers pinch-zoom + rotate), and the custom left-drag free-look / wheel handlers early-return on `pointerType==='touch'` so they don't fight OrbitControls. Canvas has `touch-none`.
- On touch, `mobileChrome` also auto-hides the play-by-play panel (`showPbpPanel=false`) alongside the scoreboard. The round selector (`.bp-rounds`) is one non-wrapping horizontally-scrolling row on all viewports (`flex-wrap:nowrap; overflow-x:auto`, hidden scrollbar, `.rt { flex-shrink:0 }`).
- **Transport** compacted (`.seek-wrap` 46px / 38px mobile, smaller play+thumb, tighter `.bp-bar` padding). On mobile `.bp-bar` has `padding-bottom: calc(9px + env(safe-area-inset-bottom))` to lift controls off the iOS home-indicator gesture zone. `.bp-controls` is constrained to its column (`width:100%; overflow-x:auto`) so the 2D/3D|cam|PLAYS|HEAT row can't overflow under the centered HUD.
- **Touch chrome polish**: seek-bar nade icons respect the util-type filter (`utilClusters` filters `utilMarkers` by `typeFilter[m.type]`, where `m.type = g.type` â€” same key the map uses) so toggling a type drops it from the bar in 2D + 3D. On touch the nade lane drops BELOW the seek track WITHOUT changing the seek bar's own height (keeping the transport row vertically centered): `.is-mobile .seek-wrap` stays 38px with `overflow:visible`, `.util-lane` is `top: calc(100% + 2px)` (hangs below), and `.bp-transport` gets `padding-bottom:30px` to reserve the room. (Earlier attempt of a tall 52px seek-wrap with rail-on-top broke the row's vertical centering â€” don't do that.) Keyboard-shortcut help (`.bp-help`) is `v-if="!mobile"`. `.bp-bar` got a near-opaque gradient + blur so the round selector/seek don't blend into the map. The sb-tools deaths toggle uses a lucide `<Skull>` (not the â˜  emoji). Tap targets bump on `.is-mobile`, with a `@media (pointer:coarse) and (min-height:600px)` tier (â‰ˆ tablets, since forced-landscape short-edge â‰¥600) making them bigger still. `isTablet` (JS: coarse + `min(innerW,innerH)â‰¥600`) keeps the scoreboard + play-by-play OPEN on tablets (phones auto-collapse both). The `.bp-more` menu pop animates via opacity+transform with `visibility 0s linear 0.15s` (delays hide so the close transition isn't cut off).
- **Mobile controls overflow menu**: on `mobile`, PLAYS + UTILITY HEATMAP â€” and in 3D the camera dock (ORBIT/TOP/CHASE/ROOF slider) â€” collapse into a 3-dot `.bp-more` menu (`v-if="mobile"`; inline buttons `v-if="!mobile"`, inline cam dock `v-if="show3d && !mobile"`) so `.bp-controls` (natural `max-content` width, NOT caged in an overflow-scroll) stays narrow. The menu is pure-CSS `:focus-within` (tabindex=0, mirrors `.bp-help`) â€” opens downward, tap-outside dismisses. ROOF gets a `.bp-more-ceil` override for a full-width row.
- **Page must not scroll**: the popout pages host the player at full viewport height. Use `h-[100dvh]` (NOT `h-screen`/100vh) â€” on mobile 100vh includes the area behind the address bar, so the container overruns the visible viewport and the page scrolls, shoving the transport bar off-screen. `pages/match-replay-popout/[matchMapId].vue` and `pages/match-3d-replay/[matchMapId].vue` both use `h-[100dvh]`.
- **Scrub pauses + resumes**: the seek scrubber captures `props.playing` on pointerdown, pauses via `onPlay()` while dragging, and resumes on release if it was playing â€” all devices (not touch-only).
- **Round selector layout**: the util-summary (buy-overlay) toggle is pinned on the left as a direct `.bp-toprow` child (moved OUT of `.bp-rounds`; `.rt` CSS rescoped from `.bp-rounds .rt` â†’ `.bp-toprow .rt` so it keeps styling). Only `.bp-rounds` (the round numbers) scrolls, with a `mask-image` left+right fade so numbers dissolve into the pinned button / the UTIL filters instead of hard-overlapping.
- **Seek + ROOF scrubbing**: native `<input type=range>` barely responds to touch (iOS only drags the 4px thumb, else the page pans). Seek: input is `pointer-events:none` (kept for thumb visual + keyboard) and a **custom pointer scrubber** lives on `.seek-wrap` (`@pointerdown=onScrubDown` â†’ window move/up, `touch-action:none`), frac from `clientX`; taps on `.mk-util-icon` excluded. The mobile-menu ROOF slider uses the same pattern (`onCeilScrubDown` on the input, `touch-action:none`) so it can be dragged, not just tapped.
- **Fullscreen is desktop-only**: the `.fs` button is `v-if="onFullscreen && !mobile"`. iOS (incl. iOS Chrome = WebKit) has NO element Fullscreen API and CSS fake-fullscreen can't hide the address bar there, so it's hidden on all touch devices. Desktop uses native `requestFullscreen`/`webkitRequestFullscreen`. The stage (`layoutRootEl`) is `bg-black` always (2D letterbox + 3D backdrop + native-fullscreen `::backdrop` all match â€” the app `--background` blue looked off here). `.fs` is 40Ã—44 to match `.spd`/`.bp-help`.
- **2D fit**: `recomputeRadarSize` normally fits the whole square map inside the stage (`min(W,H)`, letterboxes the short side). On touch (`mobileChrome`) it scales to the **band between the top score HUD and the bottom transport bar** (both float over the map): measure `.bp-hud` bottom (`topInset`) and `.bp-bar` top (`bottomInset`) vs the stage rect, `band = stageH - topInset - bottomInset`, `radarMaxPx = band - 16`, and center the square in that band via `mapTopShift = (bottomInset - topInset)/2` (applied as `top: calc(50% - mapTopShift px)`). Landscape (forced orientation) shows the whole map with side letterbox; portrait crops the sides. `minZoom2d` (computed per-fit = `min(stageW,stageH)/radarMaxPx`, â‰¤1) lets the user pinch OUT below 1Ã— to bring the whole map back when the base crops; `clampPan2d` centers the map when it's smaller than the square. Desktop (windowed + fullscreen) stays contain so a 16:9 isn't cropped. The map MUST stay square: worldâ†’radar is a fixed 1:1 projection onto a 1024Â² image (`viewBox 0 0 1024 1024`), so a non-square box misaligns the player dots â€” that's why it's a centered square, not stretched. Fullscreen element gets `bg-background` (else native fullscreen's `::backdrop` shows black behind the letterbox instead of the app blue).
- **2D pan/zoom** (`zoom2d`/`pan2dX`/`pan2dY` in ReplayViewer): the radar img + the whole SVG overlay are wrapped in a transform layer (`pan2dStyle`, `transform-origin:0 0`). Wheel + left-drag (mouse) and pinch + one-finger drag (touch) via pointer events on the `stage2dEl` square (`touch-none`). No capture / no preventDefault on pointerdown, so SVG player dots still tap-to-follow; a real drag moves far enough to suppress the click. `clampPan2d` keeps the scaled map covering the square (locked at zoom 1). Resets on view toggle. Touch drags carry **momentum**: velocity (px/ms) is tracked while dragging and on release a `requestAnimationFrame` loop glides + decays it (FRICTION 0.004/ms, killed per-axis at clamp edges); mouse releases stop immediately. The 3D viewer already glides via OrbitControls `enableDamping`. Zoom math anchors under the cursor: `pan' = v - (z1/z0)(v - pan)`.
