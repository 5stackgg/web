Mobile support for the replay stage (`components/match/ReplayViewer.vue` + `ReplayChrome.vue` + `Replay3DLite.vue`):

- **Landscape gate**: ReplayViewer keys everything off `(pointer: coarse)` = `mobileChrome` (true on phones AND tablets, false on a mouse/trackpad laptop) plus `(orientation: portrait)`. `needsRotate = coarse && portrait` shows a full-cover "rotate your device" overlay (z-[40]). No width threshold — tablets in landscape are >820px so a width gate would miss them.
- **Collapsible scoreboard**: ReplayChrome gained `mobile`, `showScoreboard`, `onToggleScoreboard` props. ReplayViewer owns `chromeScoreboardOpen` (starts collapsed on any touch device). The toggle is the first button in `.bp-sb-tools` (that strip stays mounted; `.sb-collapsed` slides it to the edge); only `.bp-score-panel` is `v-show`-hidden. The avatars/trace/deaths toggles are MAP controls so they stay visible. NOTE: the old `showScoreboard` ref + `scoreboardReserve` + the floating `.bp-score-panel` `v-if="false"` cluster in ReplayViewer are dead code — don't reuse them.
- **`.bp-chrome.is-mobile`** scoped CSS shrinks HUD / left column / scoreboard / transport for landscape phones.
- **three.js touch** (`Replay3DLite.vue`): on `(pointer: coarse)` it lowers pixelRatio cap to 1.5 + disables AA, lets OrbitControls own gestures (`enableZoom=true`, `touches: ONE=PAN, TWO=DOLLY_ROTATE` — one finger drags around, two fingers pinch-zoom + rotate), and the custom left-drag free-look / wheel handlers early-return on `pointerType==='touch'` so they don't fight OrbitControls. Canvas has `touch-none`.
- On touch, `mobileChrome` also auto-hides the play-by-play panel (`showPbpPanel=false`) alongside the scoreboard. The round selector (`.bp-rounds`) is one non-wrapping horizontally-scrolling row on all viewports (`flex-wrap:nowrap; overflow-x:auto`, hidden scrollbar, `.rt { flex-shrink:0 }`).
- **Transport** compacted (`.seek-wrap` 46px / 38px mobile, smaller play+thumb, tighter `.bp-bar` padding). On mobile `.bp-bar` has `padding-bottom: calc(9px + env(safe-area-inset-bottom))` to lift controls off the iOS home-indicator gesture zone. `.bp-controls` is constrained to its column (`width:100%; overflow-x:auto`) so the 2D/3D|cam|PLAYS|HEAT row can't overflow under the centered HUD.
- **Seek scrubbing**: native `<input type=range>` barely responds to touch (iOS only drags the 4px thumb, else the page pans). So the input is `pointer-events:none` (kept for thumb visual + keyboard) and a **custom pointer scrubber** lives on `.seek-wrap` (`@pointerdown=onScrubDown` → window move/up, `touch-action:none`), computing frac from `clientX`. Taps on `.mk-util-icon` are excluded so util clicks still work.
- **Fullscreen is desktop-only**: the `.fs` button is `v-if="onFullscreen && !mobile"`. iOS (incl. iOS Chrome = WebKit) has NO element Fullscreen API and CSS fake-fullscreen can't hide the address bar there, so it's hidden on all touch devices. Desktop uses native `requestFullscreen`/`webkitRequestFullscreen`. The stage (`layoutRootEl`) is `bg-black` always (2D letterbox + 3D backdrop + native-fullscreen `::backdrop` all match — the app `--background` blue looked off here). `.fs` is 40×44 to match `.spd`/`.bp-help`.
- **2D fit**: `recomputeRadarSize` normally fits the whole square map inside the stage (`min(W,H)`, letterboxes the short side). On touch (`mobileChrome`) it fills the **full width** (`radarMaxPx = availableW`) so the map blows up to the screen edges — taller-than-stage overflow just crops the radar's dead padding. `minZoom2d` (computed per-fit = `min(stageW,stageH)/radarMaxPx`, ≤1) lets the user pinch OUT below 1× to bring the whole map back when the base fill-width crops (landscape); `clampPan2d` centers the map when it's smaller than the square. Desktop (windowed + fullscreen) stays contain so a 16:9 isn't cropped. The map MUST stay square: world→radar is a fixed 1:1 projection onto a 1024² image (`viewBox 0 0 1024 1024`), so a non-square box misaligns the player dots — that's why it's a centered square, not stretched. Fullscreen element gets `bg-background` (else native fullscreen's `::backdrop` shows black behind the letterbox instead of the app blue).
- **2D pan/zoom** (`zoom2d`/`pan2dX`/`pan2dY` in ReplayViewer): the radar img + the whole SVG overlay are wrapped in a transform layer (`pan2dStyle`, `transform-origin:0 0`). Wheel + left-drag (mouse) and pinch + one-finger drag (touch) via pointer events on the `stage2dEl` square (`touch-none`). No capture / no preventDefault on pointerdown, so SVG player dots still tap-to-follow; a real drag moves far enough to suppress the click. `clampPan2d` keeps the scaled map covering the square (locked at zoom 1). Resets on view toggle. Zoom math anchors under the cursor: `pan' = v - (z1/z0)(v - pan)`.
